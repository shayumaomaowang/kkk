# Agent 执行引擎故障排查指南

## 问题诊断

你遇到的问题：
- LLM 返回了完整的图片提示词
- 但插件没有被继续调用来生成实际的图片

## 🔍 完整的故障排查流程

### 第一步：验证 Agent 配置

1. **访问后端调试页面**
   - 打开 http://localhost:3000/resources
   - 点击「Agent管理」选项卡

2. **检查你的 Agent 配置**
   - 找到你创建的 Agent
   - 点击「📋 查看系统提示词」展开详情
   - **关键检查点**：
     - ✅ 系统提示词中有 `⊕ 文生图-生成单张图` 的引用吗？
     - ✅ 下面是否显示「✓ 检测到的插件引用」？
     - ✅ 是否显示绿色的 `⊕ 文生图-生成单张图`（说明插件已找到）？
     - ❌ 如果显示红色的 `⚠️ 未找到` 表示插件名称不匹配

### 第二步：验证插件名称完全匹配

**常见问题**：插件名称大小写或格式不一致

| 预置插件名称 | 你的系统提示词中应该写 |
|-----------|-------------------|
| 文生图-生成单张图 | `⊕ 文生图-生成单张图` |
| 文生图-生成一组图 | `⊕ 文生图-生成一组图` |
| 豆包1.5-Pro LLM | `⊕ 豆包1.5-Pro LLM` |

### 第三步：在后端调试页面测试 Agent

1. **点击 Agent 卡片右上角的「▶ 播放」按钮**
   - 进入 Agent 测试面板

2. **输入测试数据**
   - 在"输入测试数据"框中输入：`蛋糕`

3. **点击「执行」按钮并观察浏览器控制台**

### 第四步：查看控制台日志（打开 F12 开发者工具 → Console）

**正常流程的日志应该如下**：

```
📍 解析到插件引用: "文生图-生成单张图"
📋 检测到以下插件引用: 文生图-生成单张图
🔄 调用 API: 豆包1.5-Pro LLM {body: {...}}
✅ LLM 响应: [长的图片提示词...]
🔗 准备调用插件: 文生图-生成单张图，输入内容长度: 1234
🔄 调用 API: 文生图-生成单张图 {body: {...}}
✅ 图生 API 响应: {...}
✅ 插件 文生图-生成单张图 执行成功
```

**如果出现问题的日志**：

| 日志 | 问题 | 解决方案 |
|------|------|--------|
| `⚠️ 找不到插件: 文生图-生成单张图` | 插件名不匹配 | 检查系统提示词中的插件名是否完全相同 |
| `未检测到任何插件引用` | 没有 `⊕` 符号 | 确保使用正确的 `⊕ 插件名` 格式 |
| `❌ API 调用失败` | API 认证或配置错误 | 在 API管理 标签页检查 API 配置 |
| `✅ 图生 API 响应: [object Object]` | 但没有提取到 URL | 检查响应格式是否是预期的 |

## 💡 完整的工作流程

### 系统提示词写法示例

```
# 角色
你是一个专业的图片生成助手...

## 步骤 1: 主体细节扩写
1. 接收主体名词...
2. 细节维度覆盖...

## 步骤 2: 完善提示词格式输出
生成的主体描述需直接作为...
框架：3D 电影级质感...

## 步骤 3: 图像生成
调用生成图片插件 ⊕ 文生图-生成单张图，
图像比例为1:1，只把最后完善的提示词作为提示词去生成，返回最终生图。
```

### Agent 执行流程

```
用户在首页输入 → Agent 被触发
    ↓
step 1: LLM（豆包1.5-Pro）调用
    - 输入：系统提示词 + 用户输入
    - 输出：完整的图片提示词
    ↓
step 2: 解析系统提示词中的插件引用
    - 识别所有 ⊕ 插件名
    - 按顺序构建执行队列
    ↓
step 3: 执行插件（文生图-生成单张图）
    - 输入：LLM 的输出（图片提示词）
    - 输出：图片 URL
    ↓
最终返回给用户：图片 URL
```

## 🚀 使用 PromptEditor 编辑系统提示词

### 功能

1. **⊕ 插件菜单**
   - 点击右下角的「⊕ 插件」按钮打开菜单
   - 选择要插入的插件名

2. **快速插入**
   - 点击下方蓝色的插件标签直接插入引用

3. **格式验证**
   - 保存后，系统会自动检查插件名是否匹配
   - 不匹配的插件会显示红色警告

## 📝 常见问题

**Q: 为什么插件显示「⚠️ 未找到」?**
A: 系统提示词中的插件名 `⊕ ` 后面的名称必须与 API 列表中的名称完全一致（包括连字符、大小写）。

**Q: 为什么 LLM 响应后没有继续调用图生 API?**
A: 可能原因：
1. 系统提示词中没有 `⊕ 插件名` 的引用
2. 插件名拼写错误
3. 插件没有在 Agent 的"插件/技能"列表中添加

**Q: 怎么验证图生 API 是否正常?**
A: 
1. 在后端调试页面 → API管理 标签页
2. 找到「文生图-生成单张图」
3. 点击右上角的播放按钮进行测试
4. 输入测试的 prompt 并查看返回结果

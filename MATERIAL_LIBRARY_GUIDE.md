# 素材库检索插件系统使用指南

## 📚 概念说明

### 什么是素材库检索插件？

素材库检索插件是一个**特殊类型的 Agent 插件**，它不调用外部 API，而是直接从本地素材库中检索图片。

**工作原理**：
```
LLM 输出 → 提取关键词（名词） → 从素材库中精确匹配 → 返回图片 URL
```

### 与普通 API 插件的区别

| 类型 | 数据来源 | 执行方式 | 特点 |
|------|--------|--------|------|
| **API 插件** | 外部服务（火山引擎等） | 调用网络 API | 生成新内容 |
| **素材库插件** | 本地素材库 | 本地匹配检索 | 借用现有素材 |

## 🎯 使用步骤

### 第 1 步：创建一个 Agent

1. 打开 http://localhost:3000/resources
2. 点击「Agent管理」→「新建 Agent」

### 第 2 步：基本配置

填写基本信息：
- **名称**: 如 "蛋糕设计助手"
- **主 API**: 豆包1.5-Pro LLM
- **触发源**: 自由生图
- **输出位置**: 对话窗口

### 第 3 步：添加素材库插件

在「插件/技能」部分：
- 点击「+ 📚 自由生图-参考图」
- 该插件会出现在已添加的插件列表中（蓝色标签）

### 第 4 步：编写系统提示词

**关键要点**：
1. 在系统提示词中通过 `⊕ 自由生图-参考图` 来引用素材库插件
2. 在提示词中描述期望的主体对象

**示例**：

```
## 角色
你是一个专业的蛋糕设计助手。

## 步骤 1: 分析用户需求
接收用户的蛋糕主题描述（如"婚礼蛋糕"、"儿童蛋糕"等）

## 步骤 2: 检索参考图
使用 ⊕ 自由生图-参考图 检索对应主题的蛋糕参考图片
比如说要找"蛋糕"这个素材库中有个名字叫"蛋糕"的图片

## 步骤 3: 生成设计
基于参考图片，生成最终的设计方案
```

### 第 5 步：保存并测试

1. 点击「保存」
2. 在 Agent 卡片上点击播放按钮进行测试
3. 输入测试数据，观察执行流程

## 🔍 关键词提取机制

系统提示词中引用素材库插件后，执行引擎会自动从 LLM 的输出中提取关键词。

### 支持的提取方式

#### 方式 1：【】中的关键词（推荐）
```
LLM 输出："【蛋糕】是一种甜点..."
→ 提取：蛋糕
```

#### 方式 2："一个/一只/一张" 模式
```
LLM 输出："我会为你设计一个蛋糕..."
→ 提取：蛋糕
```

#### 方式 3：直接包含关键词
```
LLM 输出："这个蛋糕设计..."
→ 提取：蛋糕
```

### 提取优先级

1. 首先查找 `【蛋糕】` 格式
2. 其次查找 `一个蛋糕` 格式
3. 最后逐字匹配常见名词

## 📊 完整的执行流程

### 示例：创建一个「参考图检索」Agent

**系统提示词**：
```
你是一个素材库检索助手。

步骤 1: 理解需求
用户会告诉你想要的物品或场景名称

步骤 2: 精确表述
用【名词】的格式表述要检索的物品
比如用户说"我要蛋糕"，你就回答"【蛋糕】"

步骤 3: 素材检索
使用 ⊕ 自由生图-参考图 从素材库中检索对应的图片

最后只返回一个名词，必须是素材库中有的
```

**执行过程**：
```
用户输入: "我要蛋糕的参考图"
    ↓
1️⃣ LLM 处理
   - 理解需求：用户要蛋糕
   - 输出：【蛋糕】
    ↓
2️⃣ 系统检测 ⊕ 自由生图-参考图 引用
    ↓
3️⃣ 提取关键词
   - 识别 【】格式
   - 提取：蛋糕
    ↓
4️⃣ 素材库检索
   - 在素材库中查找 name == "蛋糕" 的素材
   - 返回对应的图片 URL
    ↓
最终返回: 蛋糕的图片 URL
```

## 💡 最佳实践

### ✅ 好的系统提示词设计

```
## 目标
从素材库中检索指定物品的参考图片

## 步骤
1. 用户输入物品名称
2. 使用 ⊕ 自由生图-参考图 检索
3. 只返回素材库中存在的名词

## 输出格式
如果找到: 返回图片 URL
如果未找到: "素材库中未找到该物品"
```

### ❌ 避免的做法

```
❌ 不要让 LLM 生成过长的描述
   → 系统会无法准确提取关键词

❌ 不要期望检索不存在的素材
   → 系统会返回 null，无法获取图片

❌ 不要在多个步骤中使用素材库插件
   → 虽然支持，但建议单一职责
```

## 🔧 调试技巧

### 查看执行日志

打开浏览器开发者工具（F12），进入 Console 标签：

**正常执行的日志**：
```
📍 解析到插件引用: "自由生图-参考图"
📚 执行素材库插件: 自由生图-参考图
🔍 提取的关键词: ["蛋糕"]
✅ 素材库检索成功: 关键词 "蛋糕" -> /uploads/assets/xxx.jpg
```

**检索失败的日志**：
```
⚠️ 未从输出中提取到有效关键词
⚠️ 素材库中未找到匹配的图片: cake
```

### 调试步骤

1. **检查系统提示词**
   - 是否包含 `⊕ 自由生图-参考图`
   - 是否包含预期的物品名称

2. **检查 LLM 输出**
   - 打开控制台，查看 `✅ LLM 响应` 的内容
   - 确认是否包含目标物品名词

3. **检查素材库**
   - 进入「资源」→「素材库」
   - 确认是否存在对应名称的素材

4. **检查名词匹配**
   - 素材库中的 name 字段必须与提取的关键词完全一致
   - 大小写、空格、符号都要匹配

### 支持的素材库

当前仅支持：

后续可扩展：
- ⏳ `模板头图-参考图`
- ⏳ `外宣海报-参考图`
- ⏳ ...

### API 格式

素材库插件虽然显示为插件，但内部处理流程：

```javascript
// 系统提示词中的引用
⊕ 自由生图-参考图

// 内部转换
{
  type: 'material-library',
  name: '自由生图-参考图',
  endpoint: 'internal://material-library'
}

// 执行时调用
materialLibraryManager.searchByKeyword(
  '自由生图-参考图',
  '蛋糕'  // 从 LLM 输出提取
)
→ '/uploads/assets/cake.jpg'
```

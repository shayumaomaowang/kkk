# 图生图 API 集成使用指南

## 📚 新增 API 列表

你现在拥有 **6 个图片生成 API**：

### 文生图系列（2 个）
1. **文生图-生成单张图** - 从文本描述生成单张高质量图片
2. **文生图-生成一组图** - 从文本描述生成 4 张相关的多张图片

### 图生图系列（4 个新增）
3. **图生图-单张图生成单张图** - 基于 1 张参考图，生成相似风格的单张图
4. **图生图-单张图生成一组图** - 基于 1 张参考图，生成 5 张相似风格的多张图
5. **图生图-多张参考图生成单张图** - 基于 2 张参考图，融合多个元素生成单张图
6. **图生图-多张参考图生成一组图** - 基于 2 张参考图，融合多个元素生成 3 张图

## 🔄 使用场景和工作流程

### 场景 1：基于参考图生成变体

**系统提示词示例**：
```
你是一个创意设计助手。
用户会提供一个参考图片 URL。
步骤 1：分析参考图片的风格特点
步骤 2：使用 ⊕ 图生图-单张图生成单张图 生成相似风格的新图片
```

**执行流程**：
```
用户输入: "我有一张蓝色风格的设计，能生成同样风格的红色版本吗？"
    ↓
1️⃣ LLM 分析：理解用户需求和参考图风格
    → 输出：针对红色版本的详细 prompt
    ↓
2️⃣ 调用「图生图-单张图生成单张图」
    - prompt: LLM 生成的详细描述
    - image: 用户提供的参考图 URL
    → 输出：生成的新图片
    ↓
最终返回：新生成的图片
```

### 场景 2：基于多张参考图融合生成

**系统提示词示例**：
```
你是一个视觉设计专家。
步骤 1：分析两张参考图的设计元素
步骤 2：使用 ⊕ 图生图-多张参考图生成一组图 融合两种风格生成 3 张图片
```

**执行流程**：
```
用户输入: "参考图1是日式极简风，参考图2是欧式复古风，能融合两种风格生成一组图吗？"
    ↓
1️⃣ LLM 分析两种风格的特点
    → 输出：融合描述
    ↓
2️⃣ 调用「图生图-多张参考图生成一组图」
    - prompt: LLM 生成的融合描述
    - image: [参考图1, 参考图2]
    → 输出：3 张融合风格的图片
    ↓
最终返回：一组融合图片
```

### 场景 3：连续生图流程

**系统提示词示例**：
```
你是一个广告创意团队。
步骤 1：生成基础图片
使用 ⊕ 文生图-生成单张图 生成初始设计

步骤 2：基于初始设计生成变体
使用 ⊕ 图生图-单张图生成一组图 生成 5 个相似风格的变体
```

**执行流程**：
```
用户输入: "为我的咖啡品牌生成一套广告图"
    ↓
1️⃣ LLM 生成初始 prompt
    → "优雅的咖啡杯，白色背景..."
    ↓
2️⃣ 调用「文生图-生成单张图」
    → 生成初始图片 URL
    ↓
3️⃣ 调用「图生图-单张图生成一组图」
    - prompt: 可选的风格变体描述
    - image: 刚才生成的图片
    → 生成 5 张变体
    ↓
最终返回：5 张变体图片
```

## 🎯 API 参数说明

### 关键参数对比

| API | image 字段 | sequential_image_generation | 返回结果 |
|-----|-----------|---------------------------|---------|
| 文生图-生成单张图 | ❌ 无 | disabled | 1 张图 |
| 文生图-生成一组图 | ❌ 无 | auto | 4 张图 |
| 图生图-单张图生成单张图 | ✅ 单张 | disabled | 1 张图 |
| 图生图-单张图生成一组图 | ✅ 单张 | auto | 5 张图 |
| 图生图-多张参考图生成单张图 | ✅ 多张数组 | disabled | 1 张图 |
| 图生图-多张参考图生成一组图 | ✅ 多张数组 | auto | 3 张图 |

### 重要参数解释

**`sequential_image_generation`**
- `disabled` - 返回单张图片
- `auto` - 启用序列生成，返回多张相关图片

**`response_format: "url"`**
- 返回图片的 URL 而不是 base64 编码

**`size: "2K"`**
- 生成的图片分辨率（2K、1K 等）

**`stream: true/false`**
- `true` - 流式返回（适合多张生成）
- `false` - 一次性返回

**`watermark: true`**
- 添加水印

## 💡 在 Agent 中使用

### 创建一个「参考图风格迁移」Agent

1. **访问** http://localhost:3000/resources → Agent管理

2. **点击「新建 Agent」**

3. **填写信息**
   - **名称**: 参考图风格迁移
   - **主 API**: 豆包1.5-Pro LLM
   - **插件**: 勾选「图生图-单张图生成单张图」

4. **系统提示词**
   ```
   # 角色
   你是一个视觉设计专家，善于分析图片风格并进行风格迁移。

   ## 任务
   用户会提供一张参考图片 URL 和一个描述。
   你需要分析参考图片的视觉风格（色彩、构图、光影等）。
   然后生成一个详细的 prompt 来指导图生图 API 生成相同风格的新图片。

   ## 步骤
   1. 分析参考图片的风格特点（只需描述，不需要生成图片）
   2. 根据用户描述和参考图风格，生成一个详细的 prompt
   3. 使用 ⊕ 图生图-单张图生成单张图 进行生图
   4. 返回最终生成的图片 URL
   ```

5. **保存**

6. **测试**
   - 点击播放按钮
   - 输入: `https://example.com/ref-image.jpg | 生成一个同样风格的城市景观`
   - 查看生成的结果

## 📊 实际使用流程

### 示例：广告素材生成

**用户输入**:
```
我要为我的新产品「智能水杯」生成一套广告图片
```

**系统提示词**:
```
## 步骤 1: 产品理解
分析"智能水杯"产品的特点，设想最吸引人的展示角度

## 步骤 2: 生成初始图片
调用 ⊕ 文生图-生成单张图 生成一张高质量的产品图片

## 步骤 3: 生成变体
调用 ⊕ 图生图-单张图生成一组图 生成 5 张不同场景的变体

## 步骤 4: 返回结果
返回所有生成的图片 URL
```

**执行过程**:
```
1️⃣ LLM 分析产品特点
   → "科技感、现代、蓝色主色调、展示产品如何智能工作"

2️⃣ 调用文生图
   → 生成初始产品图片 URL: https://...1.jpg

3️⃣ 调用图生图
   - 参考图: https://...1.jpg
   - 提示词: "不同场景下的使用..."
   → 生成 5 张变体: https://...2-6.jpg

4️⃣ 返回给用户
   → 6 张广告素材图片
```
